# Use Qwen3-TTS for Markdown narration

The Markdown narration tool will target Qwen3-TTS as its first local text-to-speech backend because it supports English and Portuguese, is current and highly ranked on Hugging Face by downloads and likes, runs locally with CUDA, and uses the Apache-2.0 license. The default model will be `Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice` to keep downloads and GPU requirements reasonable, while allowing users to override it with the 1.7B CustomVoice model when quality matters more than runtime cost. Long documents will be chunked internally at natural text boundaries and concatenated into one WAV file, so the user-facing output remains a single narration file while staying within model limits. Model and dependency setup will remain explicit: documentation and error messages will tell users how to install dependencies and download models, but the tool will not silently download large models on normal runs. Runtime device selection will default to CUDA when available and will fail clearly rather than silently falling back to CPU after a CUDA failure; users can still choose CPU explicitly. Partial chunk files will be cleaned up after normal failures unless the user passes `--keep-temp`, which preserves them for debugging or manual recovery. XTTS-v2 remains a strong alternative, but its custom model license and older ecosystem make it a less suitable default for this repository.
